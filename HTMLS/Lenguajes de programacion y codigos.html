LOS LENGUAJES DE PROGRAMACION

  • Los lenguajes de programación son el medio por el cual los humanos se comunican con las computadoras. Son como idiomas, pero en lugar de palabras, utilizan instrucciones específicas para que las máquinas realicen tareas.
    TIPOS DE LENGUAJES DE PROGRAMACIÓN
    LENGUAJE A BAJO NIVEL
  • Los lenguajes de bajo nivel están más cerca del lenguaje que entiende directamente la computadora, el código binario (secuencias de ceros y unos). Son como hablar el idioma nativo de una máquina.
  •     Lenguaje máquina:Es el lenguaje más fundamental y cercano al hardware. Consiste en una secuencia de ceros y unos que la CPU interpreta directamente. Cada combinación de bits representa una instrucción específica para la máquina.
    Los lenguajes a bajo nivel pueden ser:
  •     Lenguaje máquina:Es el lenguaje más fundamental y cercano al hardware. Consiste en una secuencia de ceros y unos que la CPU interpreta directamente. Cada combinación de bits representa una instrucción específica para la máquina.
  •     Ejemplo:Una instrucción simple como "sumar dos números" se representaría como una larga secuencia de ceros y unos.

  •     Lenguaje ensamblador:Es un paso por encima del lenguaje máquina. Utiliza códigos mnemotécnicos (palabras abreviadas) para representar las instrucciones, lo que lo hace un poco más legible.
        Por ejemplo, en lugar de escribir una larga secuencia de bits para sumar dos números, podrías escribir algo como "ADD A, B".
    CARACTERISTICAS DE LOS LENGUAJES A BAJO NIVEL:
  •     Mayor control sobre el hardware: Permiten optimizar al máximo el rendimiento y utilizar recursos de forma eficiente.

  •     Dependencia de la arquitectura: Los programas escritos en estos lenguajes suelen estar vinculados a una arquitectura específica de procesador.

  •     Dificultad de aprendizaje: Requiere un conocimiento profundo de la arquitectura de la computadora.

  •     Menor productividad:La programación en estos lenguajes es más lenta y propensa a errores.
    USOS TIPICOS:
  • Desarrollo de sistemas operativos.

  • Controladores de dispositivos.

  • Aplicaciones que requieren un alto rendimiento y un control preciso sobre el hardware

    LENGUAJE A ALTO NIVEL
  • Los lenguajes de alto nivel están diseñados para ser más fáciles de entender y usar por los programadores. Se asemejan más al lenguaje humano y abstraen la complejidad del hardware.
    Los lenguajes a alto nivel pueden ser:
  •     Estructurados: Siguen una estructura lógica y ordenada, utilizando bloques de código para organizar las instrucciones. Ejemplos: Pascal, C

  •     Orientados a objetos: Se basan en el concepto de objetos, que encapsulan datos y comportamiento. Ejemplos: Java, C++, Python.
    CARACTERISTICAS DE LOS LENGUAJES A ALTO NIVEL:
  •     Abstracción: Ocultan la complejidad del hardware, permitiendo al programador concentrarse en la lógica del problema.

  •     Legibilidad:Utilizan palabras clave y sintaxis más cercanas al lenguaje natural.

  •     Portabilidad:Los programas escritos en lenguajes de alto nivel pueden ejecutarse en diferentes tipos de computadoras.

  •     Facilidad de mantenimiento: Los programas son más fáciles de entender y modificar.
    ALGUNOS ELEMENTOS BASICOS DE UN LENGUAJE DE PROGRAMACIÓN SON :
  •     Identificadores: Nombres que se asignan a variables, funciones, etc.

  •     Constantes: Valores que no cambian durante la ejecución del programa.

  •     Instrucciones: Órdenes que el programa ejecuta.

  •     Comentarios: Texto que explica el código y facilita su comprensión.

  •     Operadores: Símbolos que realizan operaciones matemáticas o lógicas.

CODIGO FUENTE

    ¿QUE ES EL CODIGO FUENTE?
  •     Es el texto que escribe un programador en un lenguaje de programación (como Python, Java, C++, etc.). Es la forma humana-legible de un programa y contiene la lógica y las instrucciones que el programa debe seguir.

    ALGUNAS CARACTERISTICAS DEL CODIGO FUENTE SON:
  •     Escrito en un lenguaje de programación: Puede ser Python, Java, C++, etc.

  •     Contiene la lógica del programa: Las instrucciones precisas que la computadora debe seguir para realizar una tarea.

  •     Es editable: Puedes modificar el código fuente para corregir errores o añadir nuevas funcionalidades.

  •     Función:Contiene las instrucciones detalladas que el programa debe seguir para realizar una tarea específica.

    • CODIGO OBJETO

      ¿QUE ES EL CODIGO OBJETO?
    •     Es la versión traducida del código fuente a un lenguaje que la computadora puede entender mejor, pero aún no es directamente ejecutable. Es como si tomaras el plano de la casa y lo convirtieras en una lista detallada de materiales y pasos a seguir, pero aún no tienes los materiales ni has empezado a construir.

      ALGUNAS CARACTERISTICAS DEL CODIGO OBJETO SON:
  •     Es una versión traducida del código fuente: Ha sido transformado a un lenguaje de máquina (ceros y unos) o a un lenguaje intermedio que la computadora puede entender directamente.

  •     No es ejecutable directamente: Necesita ser combinado con otras partes para formar un programa completo.

  •      Contiene las instrucciones en un formato que la computadora puede procesar.

  •     Función: Sirve como un puente entre el código fuente y el código ejecutable. Es un paso necesario en el proceso de compilación.

    • CODIGO EJECUTABLE

      ¿QUE ES EL CODIGO EJECUTABLE?
    •     Es la versión final del programa, listo para ser ejecutado por la computadora. Es como la casa construida, lista para ser habitada.

      ALGUNAS CARACTERISTICAS DEL CODIGO EJECUTABLE SON:
  •     Es la versión final del programa: Contiene todas las instrucciones necesarias para que la computadora realice la tarea.

  •     Es ejecutable directamente: Al hacer doble clic en el archivo, el programa se inicia.

  •     Está en un formato que el sistema operativo puede cargar en la memoria y ejecutar.

  •     Función: Contiene todas las instrucciones necesarias para que la computadora realice la tarea especificada en el código fuente.

    • EL PROCESO DE COMPILACION Y ENLACE

        Para pasar del código fuente al código ejecutable, se necesitan dos procesos principales:


  •     Compilación: El compilador toma el código fuente y lo traduce a código objeto. En este proceso, se verifica la sintaxis del código y se generan las instrucciones correspondientes en lenguaje máquina.

  •     Enlace: El enlazador combina el código objeto con las bibliotecas necesarias para crear el programa ejecutable. Las bibliotecas son conjuntos de código precompilado que proporcionan funcionalidades adicionales al programa.

    • TIPOS DE TRADUCTORES

        Los traductores son programas que convierten el código fuente en código máquina. Los principales tipos son:


  •     Compiladores: Son traductores que realizan su tarea de forma global, esto es, en un único proceso se analiza todo el programa fuente, se genera el código objeto respectivo y se almacena el resultado. Todo esto se podrá llevar a cabo en caso de que el programa fuente esté correctamente escrito de acuerdo con las normas del lenguaje de programación correspondiente; en caso de que no sea así, no se generará el código objeto y se mostrarán uno o varios mensajes de error que pueden ayudar a la persona encargada de la programación a corregir estos errores. El código objeto generado como resultado de la compilación, dependiendo del tipo de compilador, se podrá ejecutar directamente o puede que se precisen otros pasos antes de que el programa sea ejecutable, como los pasos de ensamblado, enlazado y carga. Una vez obtenido el código ejecutable, este se podrá ejecutar tantas veces como se desee sin necesidad de tener que volver a realizar el proceso de compilación.

  •     Intérpretes: A diferencia de lo que ocurre con los compiladores, estos simultanean el proceso de traducción y el de ejecución. La forma de trabajo de los intérpretes es la siguiente: analizan bloques del programa fuente, generan el código objeto correspondiente y lo ejecutan, luego repiten este proceso hasta que acaba el programa. En la etapa de ejecución simplemente se pasa el control al código objeto generado y se espera a que termine su ejecución. De esta forma, cada fragmento de código fuente solo se almacena provisionalmente.

    • EL CODIGO INTERMEDIO PARA SU GENERACION EN MAQUINAS VIRTUALES

        La generación de código intermedio es una etapa fundamental en el proceso de compilación de un lenguaje de programación. Este código, que se encuentra a un nivel de abstracción entre el lenguaje fuente y el código máquina, desempeña un papel crucial en la portabilidad y eficiencia de las aplicaciones.


      CARACTERISTICAS PRINCIPALES
          - Independencia de la arquitectura:

               o El código intermedio se diseña para ser independiente de la arquitectura de la máquina objetivo. Esto significa que el mismo código intermedio puede ser ejecutado en diferentes plataformas (sistemas operativos, procesadores) sin necesidad de recompilar el programa fuente.

               o Esta característica es clave para lograr la portabilidad de las aplicaciones, ya que permite ejecutar un mismo programa en diversos dispositivos sin modificaciones significativas.
          - Optimización:

               o El código intermedio ofrece una representación del programa que facilita la aplicación de técnicas de optimización.

               o Estas optimizaciones pueden realizarse a nivel de código intermedio, lo que permite mejorar el rendimiento del programa antes de generar el código máquina específico para una determinada plataforma.

               o Algunas optimizaciones comunes incluyen la eliminación de código muerto, la propagación de constantes, y la reordenación de instrucciones.
          - Abstracción:

               o El código intermedio proporciona un nivel de abstracción más alto que el código máquina, lo que facilita la comprensión y el análisis del programa.

               o Esta abstracción también simplifica la tarea de construir herramientas de desarrollo, como depuradores y perfiladores.
          - Representación:

               o El código intermedio se representa típicamente como una secuencia de instrucciones o como un grafo de flujo de control.

               o Las instrucciones del código intermedio suelen ser más simples y menos numerosas que las instrucciones del código máquina, lo que facilita su generación y manipulación.
          - Máquina virtual:

               o El código intermedio está diseñado para ser ejecutado por una máquina virtual.

               o Una máquina virtual es un programa que emula el comportamiento de una computadora real, interpretando las instrucciones del código intermedio y ejecutando las operaciones correspondientes en el hardware subyacente.

               o La máquina virtual proporciona un entorno de ejecución aislado para el código intermedio, lo que aumenta la seguridad y la fiabilidad de las aplicaciones.
      LAS VENTAJAS DE UTILIZAR EL CODIGO INTERMEDIO Y LAS MAQUINAS VIRTUALES SON:
    •     Portabilidad: Las aplicaciones pueden ejecutarse en diferentes plataformas sin recompilación.

    •     Seguridad: El entorno aislado de la máquina virtual protege al sistema operativo y al hardware subyacente de posibles errores en el código.

    •     Flexibilidad: Las máquinas virtuales permiten ejecutar múltiples sistemas operativos y aplicaciones en un mismo hardware.

    •     Eficiencia: Las optimizaciones a nivel de código intermedio pueden mejorar significativamente el rendimiento de las aplicaciones.

      ALGUNOS EJEMPLOS DE LENGUAJE DE PROGRAMACION QUE UTILIZAN UN CODIGO INTERMEDIO SON:
    •     Java: El bytecode de Java es un ejemplo clásico de código intermedio que se ejecuta en la Máquina Virtual de Java (JVM).

    •     Python: Aunque Python se interpreta directamente, algunos implementaciones generan un bytecode intermedio para optimizar la ejecución.
