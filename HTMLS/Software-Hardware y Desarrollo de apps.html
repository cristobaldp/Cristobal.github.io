Hardware:

Son los componentes físicos de un ordenador: el procesador, la memoria RAM, el disco duro, la tarjeta gráfica, el teclado, el ratón, etc. Son las partes tangibles que puedes tocar.

Software:

Son los programas que le indican al hardware qué hacer. Esto incluye el sistema operativo (como Windows, macOS o Linux), las aplicaciones (como Word, Excel, Chrome), y todos los demás programas que usamos.

¿Cómo interactúan?

El software envía instrucciones al hardware, y este las ejecuta. Por ejemplo, cuando escribes un correo electrónico, el software de tu procesador de texto envía instrucciones a la memoria RAM para almacenar el texto, al disco duro para guardarlo y a la tarjeta gráfica para mostrarlo en la pantalla. ¿Por qué es importante la compatibilidad entre hardware y software?

¿Qué es la Ingeniería del Software?

La ingeniería del software es la aplicación de un enfoque sistemático, disciplinado y cuantificable al desarrollo, operación y mantenimiento 1 de software. Es decir, 2 es la disciplina que se encarga de crear software de alta calidad de manera eficiente y confiable.

Fases del Desarrollo de una Aplicación Informática

El desarrollo de software sigue generalmente un proceso que involucra varias fases:

  • Análisis de requisitos: Se define qué es lo que el software debe hacer, cuáles son las necesidades del usuario y las restricciones del sistema.
  • Diseño: Se crea una estructura general del software, definiendo los componentes, la arquitectura y la interfaz de usuario.
  • programación: Se escribe el código fuente de la aplicación utilizando un lenguaje de programación específico.
  • Pruebas: Se verifica que el software funcione correctamente y cumpla con los requisitos establecidos.
  • Explotación: Se despliega el software en el entorno de producción para que los usuarios puedan utilizarlo.
  • Mantenimiento: Se realizan correcciones de errores, mejoras y adaptaciones del software a lo largo de su ciclo de vida.

    • Estas actividades fueron denominadas por Pressman como actividades estructurales, pero para el desarrollo de software son necesarias otras actividades denominadas según Pressman como actividades sombrilla:

  • Seguimiento y control del proyecto de software: Se debe evaluar el proyecto periodicamente con el fin de comprobar si se esta siguiendo el plan estaablecido
  • Administración del riesgo: Consiste en evaluar los riesgos que pueden afectar al resuslado del proyecto o a la calidad del producto
  • Aseguramiento de la calidad del software: Se definen y ejecutan las actividades necesarias para garantizar la calidad del software
  • Medición: Consiste en definir y reunir las mediciones del proceso, proyecto y prodcto para ayudar al equipo a entregar el software que satisfaga las necesidades del cliente
  • Administración de la configuracion del software: Se administran los efectos del cambio a lo largo del proceso de desarrollo de software
  • Administración de la reutilización: Se establecen los criterios para poder usar productos del trabajo en proyectos posteriores y los mecanismos para que esto se pueda llevar a la practica, es decir, para obtener componentes reutilizables.
  • Preparación y producción del producto del trabajo: Se agrupan las actividades necesarias para crear productos del trabajo, como modelos, documentos, registros, formatos, etc...

    • Roles que Intervienen en el Desarrollo de Software

  • Jefe de proyecto: Planifica todo el proyecto, su puesta en marcha, su ejecucción, seguimiento, control y cierre.
  • Expertos del dominio: Personas que trabajan para la empresa para la que hay que desarrollar la aplicación, son los que conocen con detalle las necesidades que debe cubrir el software.
  • Analista: Se encarga del analisis, es decir, crea un modelo claro y consistente que se corresponda con la vision del problema que se ha acordado con la ayuda de los expertos del dominio
  • Arquitecto: Se encarga de definir las lineas maestras del diseño estableciendo la arquitectura del sistema .
  • Diseñador: Diseña las partes del sistena que implementan los requisitos hasta un nivel de detalle suficiente para ayudar con la tarea de programación.
  • Programador: Escribe el código fuente de la aplicación.
  • Probador: Realiza pruebas para asegurar la calidad del software.
  • Encargado de la implantación: Se encarga de que se realice el trabajo de empaquetado del software y de su instalación y también se encarga de la configuaración de este.
