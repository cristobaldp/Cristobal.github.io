¿QUE ES EL SOFTWARE?

Conjunto de rogramas que le indican a una computadora cómo realizar tareas específicas.

PARTES DEL SOFTWARE

-Programas: Instrucciones para la computadora.

-Aplicaciones: Programas para tareas específicas.

-Sistema operativo: El jefe de la computadora.

-Lenguajes de programación: Los idiomas de las computadoras.

-Algoritmos: Los pasos lógicos dentro de un programa.

-Compilación e interpretación: Formas de ejecutar programas.

-Depuración: Corregir errores.

Tipos de software: Sistema, aplicación y programación.

FASES DE DESARROLLO DEL SOFTWARE: Planificación:

Definición de objetivos: Se establecen los objetivos claros y medibles del proyecto.

Análisis de requisitos: Se identifican y documentan las necesidades y expectativas del usuario.

Planificación del proyecto: Se crea un plan detallado con hitos y plazos.

Diseño:

Diseño arquitectónico: Se define la estructura general del software, incluyendo los componentes principales y su interacción.

Diseño de interfaz: Se crea el diseño de la interfaz de usuario (UI) y la experiencia de usuario (UX).

Diseño de base de datos: Si es necesario, se diseña la estructura de la base de datos para almacenar la información.

Desarrollo:

Codificación: Se escribe el código fuente utilizando los lenguajes de programación seleccionados.

Integración: Se combinan los diferentes componentes del software.

Construcción: Se crea una versión ejecutable del software.

Pruebas:

Pruebas unitarias: Se verifican las funcionalidades individuales del software.

Pruebas de integración: Se evalúa la interacción entre los diferentes componentes.

Pruebas de sistema: Se verifica el funcionamiento del software completo.

Pruebas de aceptación: Se asegura que el software cumple con los requisitos del usuario.

Implementación:

Despliegue: Se instala el software en el entorno de producción.

Migración de datos: Si es necesario, se transfieren los datos existentes al nuevo sistema.

Capacitación: Se capacita a los usuarios en el uso del software.

Mantenimiento:

Corrección de errores: Se solucionan los problemas que surgen después de la implementación.

Mejora de rendimiento: Se optimiza el software para mejorar su velocidad y eficiencia.

Adición de nuevas funcionalidades: Se incorporan nuevas características al software según las necesidades del usuario.

MODELOS DE CICLO DE VIDA:

Modelo en cascada

El modelo en cascada es una metodología de desarrollo de software que sigue un enfoque secuencial y lineal. Cada fase del proyecto (análisis, diseño, implementación, pruebas, etc.) se completa por completo antes de pasar a la siguiente. Este modelo es rígido y menos flexible ante cambios, ya que asume que todos los requisitos están definidos al inicio del proyecto. Aunque es fácil de entender, puede no ser adecuado para proyectos complejos o con requisitos cambiantes, donde metodologías más ágiles como Scrum o Kanban suelen ser preferibles.

Modelo en V

El modelo en V es una extensión del modelo en cascada que enfatiza la verificación y validación en cada fase del desarrollo de software. Tiene forma de "V" debido a la correspondencia entre las fases de diseño (brazo izquierdo de la V) y las fases de prueba (brazo derecho). Cada etapa de diseño tiene una etapa de prueba asociada para asegurar que el producto cumple con los requisitos. Este modelo busca garantizar la calidad del software desde las primeras etapas del desarrollo, pero sigue siendo un enfoque secuencial y menos flexible que las metodologías ágiles.

Modelo incremental:

El modelo incremental divide el desarrollo de software en pequeñas iteraciones o incrementos. Cada incremento entrega una parte funcional del producto final. Los requisitos se priorizan y se implementan gradualmente. Esto permite obtener feedback del cliente más temprano y adaptarse a los cambios con mayor facilidad. Al finalizar cada incremento, se obtiene una versión del producto que puede ser entregada o utilizada como base para el siguiente incremento. Este modelo es más flexible y adaptable que el modelo en cascada, permitiendo una entrega más rápida de valor al cliente.

Modelo iteractivo

:El modelo iterativo es una metodología de trabajo que se basa en la repetición de ciclos cortos de desarrollo. En cada ciclo, se planifica, diseña, implementa y evalúa una parte del proyecto. Este enfoque permite obtener resultados parciales de forma rápida y ajustar el proyecto sobre la marcha, basándose en la retroalimentación recibida. Los ciclos se repiten hasta alcanzar el producto final, lo que garantiza una mayor flexibilidad y adaptación a los cambios.

Modelo en espiral:

El modelo en espiral es una metodología de desarrollo de software que combina elementos de los modelos secuenciales (como el modelo en cascada) con la naturaleza iterativa de los modelos de prototipado. Se representa como una espiral, donde cada vuelta representa una fase del desarrollo. En cada vuelta se realizan actividades de planificación, análisis de riesgos, ingeniería y evaluación. Este enfoque permite una gestión más efectiva de los riesgos y una mayor flexibilidad para adaptarse a los cambios en los requisitos del proyecto, haciendo que el producto final sea más robusto y se ajuste mejor a las necesidades del cliente.

METODOLOGÍAS EN EL DESARROLLO DE SOFTWARE:

Hay dos tipos de metodologias

Metodologias tradicionales:

Tipos de metodoligias tradicionales

Cascada:

- Ventajas: Estructura clara, documentación detallada, adecuada para proyectos estables.

- Desventajas: Rigidez, tiempo de espera, dificultad para adaptarse a cambios y mayor riesgo de errores.

Prototipado:

- Ventajas: Validación de ideas, detección de problemas, mejora de comunicación y reducción de riesgos.

- Tipos: Alta fidelidad (detallado) y baja fidelidad (básico).

Metodologias Agiles:

Priorizan la adaptación al cambio, colaboración entre equipos y entregas frecuentes de software funcional.

Tipos de metodologias agiles:

Scrum: Divide el proyecto en sprints (ciclos cortos) y fomenta pruebas continuas.

- Ventajas: Entrega regular de valor, mejora de calidad, seguimiento del progreso por el cliente.

Kanban:Visualiza el flujo de trabajo en tableros, permite una gestión flexible y mejora continua.

- Ventajas: Adaptación a cambios, reducción de errores, personalización para cada equipo.

Extreme Programming (XP): Enfocado en calidad del código, pruebas frecuentes y refactorización constante.

- Ventajas: Feedback temprano, ciclos cortos, satisfacción del cliente, buena comunicación del equipo.

Desarrollo Impulsado por Características (FDD): Entrega de características completas y funcionales mediante tareas desglosadas.

- Ventajas: Diseños de alto nivel, adecuado para proyectos grandes, iteraciones cortas con fechas definidas.
